## Question

What format should the graphs and charts take in the analytics report?

Options to decide between:
- **JSON chart data** stored in `report` JSONB, rendered by a frontend charting lib (e.g. Recharts — already using React)
- **HTML string** generated by the analytics agent, stored and rendered in an iframe or `dangerouslySetInnerHTML`
- **SVG strings** embedded directly
- **No charts in the report itself** — just structured numbers; frontend builds its own charts from the data

Blocked on ticket 01 because the chart axes depend on knowing which metrics exist.

## Resolution

No charts stored in the report. The `report` JSONB contains structured numbers only; the frontend builds its own visualizations from those values.

**`report` JSONB shape (required fields):**

```ts
{
  view_count: number;
  like_count: number;
  comment_count: number;
  share_count: number;
  favorites_count: number;
  engagement_rate: number; // (likes+comments+shares+favorites) / views
}
```

No narrative text, no week-over-week delta, no chart config. The analytics agent outputs this flat object and nothing else for the report field. The frontend is responsible for deciding how to display/chart the data.
